# WebDAV Sync — синхронизация Obsidian через свой VPS

Свой плагин синхронизации хранилища между устройствами (**Linux / Android / iOS**) через
**твой собственный сервер** по протоколу WebDAV. Без подписок, без чужих облаков, данные
лежат обычными файлами на твоём VPS.

Идея настройки ровно та, что ты хотел:

- **На ПК:** ввёл адрес, юзера и пароль → кнопка «Подключиться и настроить» — плагин сам
  создаёт папку на сервере и заливает хранилище.
- **На телефоне:** «просто подключить» — вставил один код с ПК → устройство само
  настроилось и подтянуло заметки.

## Почему WebDAV, а не git/SFTP

Внутри мобильного Obsidian (iOS/Android) нельзя открыть SSH/SFTP — нет сырых сокетов.
Работает только то, что ходит по HTTP. Поэтому «юзер + IP + пароль» на практике = **WebDAV**
поверх HTTPS: единственный транспорт, который одинаково работает и на десктопе, и на
телефоне (через `requestUrl` Obsidian, минуя CORS). Надёжно, просто, бесплатно.

## 1. Сервер на VPS (один раз)

```bash
cd server
cp .env.example .env        # или просто запусти setup.sh — он спросит всё сам
chmod +x setup.sh
./setup.sh
```

Скрипт спросит юзера, пароль и домен, поднимет два контейнера (`webdav` + `caddy`) и в конце
напечатает данные для плагина.

- **С доменом** (рекомендуется, работает на iOS): заведи A-запись `vault.example.com → IP`
  VPS, открой порты **80** и **443**. Caddy сам получит HTTPS-сертификат Let's Encrypt.
- **Без домена** (быстрый тест, только десктоп/Android): раскомментируй `ports: 8080:80`
  в `docker-compose.yml`, оставь `DOMAIN` пустым, подключайся на `http://<ip>:8080` с
  выключенным HTTPS. iOS такой сервер не примет.

Заметки на сервере лежат как обычные файлы в `server/data/` — удобно бэкапить.

## 2. Плагин в Obsidian

Собери и положи в хранилище:

```bash
npm install
npm run build           # создаёт main.js
```

Скопируй `manifest.json`, `main.js`, `styles.css` в папку
`<хранилище>/.obsidian/plugins/khannan-webdav-sync/`, включи плагин в
*Settings → Community plugins*.

> `main.js` не коммитится в репозиторий (см. `.gitignore`) — собирается из исходников.
> `data.json` тоже игнорируется: в нём хранится **пароль** от WebDAV.

## 3. Первичная настройка (на ПК)

*Settings → WebDAV Sync*:

1. Адрес, порт, HTTPS, пользователь, пароль (из вывода `setup.sh`).
2. «Проверить» — убедиться, что сервер отвечает.
3. «Подключиться и настроить» — плагин создаёт папку на сервере и делает первую
   синхронизацию.

## 4. Подключить телефон

На ПК: задай (по желанию) секретную фразу → «Создать код» → код скопируется в буфер.
На телефоне: тот же плагин → поле «Вставить код с ПК» → (та же фраза, если шифровал) →
«Подключить это устройство». Готово — заметки подтянутся сами.

Код бывает двух видов: `WDS1:` (открытый, содержит пароль — держи в секрете) и `WDSX:`
(зашифрован секретной фразой — безопасно передавать).

## Как работает синхронизация и безопасность данных

- Двусторонняя синхронизация с **индексом-предком** (сравнение по содержимому, SHA-256).
- **Ничего не теряется.** Если один файл изменили на двух устройствах по-разному —
  сохраняются **обе** версии: локальная остаётся основной, удалённая ложится рядом копией
  `имя (conflict <устройство> <время>).md` на всех устройствах.
- Удаление переносится только если файл не менялся на другой стороне (иначе файл
  восстанавливается — приоритет у сохранности).
- Синхронизация: по кнопке, по таймеру (интервал в настройках) и через несколько секунд
  после правок.

## Что синхронизируется

Заметки и вложения хранилища. Служебная папка `.obsidian` (тема, горячие клавиши,
`workspace.json`, **и данные этого плагина с паролем**) намеренно **не** синхронизируется.

## Ограничения / на будущее

- WebDAV на мобильных использует `requestUrl` Obsidian с методами PROPFIND/MKCOL/PUT/DELETE.
  На десктопе и Android это стабильно; на iOS проверь на реальном устройстве (тот же подход
  используют популярные плагины, но железо стоит протестировать).
- Нет синхронизации `.obsidian`-конфига (можно добавить опцией позже).
- Нет удаления «в корзину» на стороне сервера — удаления применяются напрямую.

## Разработка

```bash
npm run dev     # пересборка в watch-режиме
npm run build   # проверка типов + продакшн-сборка
```

Структура: `src/webdav` — WebDAV-клиент, `src/sync` — движок и типы, `src/settings*` —
настройки и UI, `src/main.ts` — точка входа, `server/` — установка сервера.
